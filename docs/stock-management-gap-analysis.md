## 六、数据模型差距

### 6.1 库存粒度

**Grocy：** `stock_entry` 是最小单位
```
stock_entry
├── product_id
├── amount (数量)
├── location_id (位置)
├── price (价格)
├── best_before_date (过期日)
├── purchased_date (购买日)
├── open (开封状态)
├── opened_date (开封日期)
└── shopping_location_id (商店)
```
每笔入库独立记录，同产品可在不同位置各有一份。

**HomeHub：** `inv_items` + `inv_item_batches`
```
inv_items (物品 — 聚合粒度)
├── quantity (总数量)
├── location_id (单一位置)
├── shop / purchasePrice / avgPrice
└── inv_item_batches (批次 — 按过期日分组)
    ├── quantity
    ├── expiry_date
    └── location_id (可选)
```
物品级别只有一个位置，批次可以有不同位置但 UI 未充分利用。

### 6.2 消费排序

**Grocy：** 已开封 → 最早过期 → 最早入库
**HomeHub：** 最早过期 → 最早入库（缺少开封优先）

### 6.3 交易类型

**Grocy：** purchase / consume / inventory-correction / stock-opening
**HomeHub：** add / consume / transfer / adjust（缺少 stock-opening）

## 九、编辑功能详细设计

### 需要编辑的字段

**基本信息（必填）：**
- name（名称）
- type（类型/分类）
- unit（单位）

**基本信息（可选）：**
- categoryId（分类）
- locationId（位置）
- brand（品牌）
- barcode（条码）
- notes（备注）
- image（图片）
- spec（规格）

**库存相关：**
- minStock（最低库存）
- shop（默认商店）

**价格相关：**
- purchasePrice（购买价格）
- currency（货币）

**日期相关：**
- purchaseDate（购买日期）
- expiryDate（过期日期）

**高级：**
- customFields（自定义字段）

### 不应编辑的字段

- quantity（通过入库/消费/调整修改）
- avgPrice / minPrice / maxPrice（自动计算）
- lastPrice（自动更新）
- createdAt / updatedAt（自动维护）
- familyId（不可变）
- productId（关联关系不可变）

### 实现位置

1. **StockList.vue** — "更多操作"菜单的"编辑"项 → 打开编辑弹窗
2. **ItemDetailModal.vue** — 信息 tab 右上角加"编辑"按钮 → 切换为编辑模式
3. **ItemDetail.vue** — 操作栏加"编辑"按钮 → 打开编辑弹窗

### API 已支持

`PUT /stock/items/:id` 的 `UpdateItemDto` 已包含所有可编辑字段，`stock.service.ts` 的 `update()` 方法已实现完整逻辑（包括 opened 状态的特殊处理）。
